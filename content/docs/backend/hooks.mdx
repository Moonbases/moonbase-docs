---
title: Backend Hooks
description: Run logic when routes are ready or on shutdown.
---

The backend loader exposes two hook points:

<TypeTable
  type={{
    'hooks.onRoutesReady(callback)': {
      type: '(callback) => void',
      description: 'Runs after routes are registered.',
      required: true
    },
    'hooks.onShutdown(callback)': {
      type: '(callback) => void',
      description: 'Runs when the API is shutting down.',
      required: true
    }
  }}
/>

<Callout type="info" title="When to use hooks">
  Use them for actions that must wait until routes are registered or for cleanup before the API shuts down.
</Callout>

```js
module.exports = {
  name: 'cleanup-example',
  register(ctx) {
    ctx.hooks.onRoutesReady(() => {
      ctx.logger.info('routes ready');
    });

    ctx.hooks.onShutdown(() => {
      ctx.logger.info('api shutting down');
    });
  }
};
```
